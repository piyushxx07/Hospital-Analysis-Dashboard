# 📖 Project Overview

Hospitals generate massive amounts of operational data every day, including patient records, doctor information, appointments, billing transactions, medicine inventory, staff records, and hospital resources.

When these datasets are maintained separately, identifying operational trends and making timely decisions becomes difficult.

This project demonstrates an **End-to-End Hospital Analytics Dashboard** built in **Power BI**, transforming multiple Excel datasets into meaningful healthcare insights through interactive visualizations, KPI reporting, and business intelligence dashboards.

---

# 🎯 Business Problem

Healthcare administrators need answers to questions like:

- How many patients are admitted every month?
- Which departments generate the highest revenue?
- Which doctors treat the most patients?
- What is the current bed occupancy?
- Which medicines require replenishment?
- How satisfied are patients with hospital services?

Without centralized reporting, answering these questions requires manual reporting across multiple datasets.

This dashboard provides a single source of truth for monitoring hospital operations and supporting data-driven healthcare decisions.

# 🔄 Data Preparation

The raw Excel datasets were transformed using **Power Query** before building the dashboard.

### Data Cleaning Steps

✔ Removed unnecessary columns

✔ Renamed inconsistent fields

✔ Corrected data types

# 🧩 Data Model

The dashboard follows a relational data model connecting patients, doctors, appointments, billing, medicine inventory, departments, and hospital resources.

The optimized model enables accurate KPI calculations, faster dashboard performance, and seamless cross-filtering between reports.
